# 项目定位

**项目名：PyType / Python打字闯关**

一句话：

**一个把打字练习和轻小游戏结合起来的纯前端工具，帮助 Python 初学者熟悉键盘输入、代码字符和基础编程词汇。**

------

# 一、MVP目标

第一版只验证 3 件事：

1. 用户愿不愿意打开就练
2. 游戏化练习能不能提升打字兴趣
3. 用户能不能在练打字时顺便熟悉 Python 常见输入内容

所以第一版必须满足：

- 纯前端
- 双击即可运行
- 手机可打开
- 不依赖后端
- 有即时反馈
- 有分数和进度
- 有一点小游戏感

------

# 二、目标用户

## 核心用户

- Python 初学者
- 键盘不熟练的小白
- 打代码很慢的新手
- 对括号、符号、缩进输入不熟的人

## 核心痛点

- 键盘盲打能力差
- 输入英文单词慢
- 打括号、引号、冒号很不顺
- 代码输入经常出错
- 学 Python 时因为打字太慢而烦躁

------

# 三、项目核心思路

这个项目不要做成传统打字网站，而要做成：

**“面向 Python 初学者的代码输入小游戏”**

也就是说，练习内容不只是英文句子，而是：

- Python 单词
- 常见代码片段
- 符号输入
- 缩进节奏
- Python 关键字

这样用户会觉得：

**我不是在机械练打字，而是在为写代码做准备。**

------

# 四、MVP玩法建议

我建议第一版做 **1 个主玩法 + 2 个辅玩法**。

------

## 主玩法：代码雨打字闯关

屏幕上会不断掉落单词或代码片段，用户要在输入框中正确输入，把它“消除”。

### 掉落内容可以是：

- print
- input
- list
- if
- else
- for
- while
- return
- `()`
- `[]`
- `{}`
- `:`
- `==`
- `!=`

### 游戏体验

- 打对一个，得分
- 打错会扣血或减时间
- 连续打对有连击
- 到一定分数过关

这个玩法最适合做第一版，因为：

- 游戏感最强
- 规则简单
- 好开发
- 用户容易上手

------

## 辅玩法1：单词冲刺

给一个单词或短代码，用户尽快打完。

例如：

- `print`
- `input`
- `for`
- `name`
- `range`

统计：

- 用时
- 正确率
- WPM 简版

------

## 辅玩法2：符号训练

专门练这些最让小白头疼的输入：

- `()`
- `[]`
- `{}`
- `""`
- `''`
- `:`
- `=`
- `==`
- `!=`

让用户快速熟悉代码常见符号。

------

# 五、MVP功能范围

## 必须有的功能

### 1. 首页

首页展示：

- 项目名称
- 项目介绍
- 开始游戏按钮
- 玩法选择
- 学习记录摘要

比如：

标题：
 **Python打字闯关**

副标题：
 **边练打字，边熟悉 Python 常见单词和代码输入。**

按钮：

- 开始闯关
- 单词冲刺
- 符号训练

------

### 2. 游戏主界面

显示内容：

- 当前模式
- 分数
- 血量 / 生命值
- 当前关卡
- 掉落的目标词
- 输入框
- 暂停按钮

主界面一定要简单，重点放在“输入”和“反馈”。

------

### 3. 即时判定

用户输入后：

- 完全正确：目标消失，得分
- 错误：提示错误，扣血或不给分
- 连续正确：显示连击数

这个即时反馈是游戏感核心。

------

### 4. 关卡系统

第一版不用做太复杂，但至少分 3 类：

- 基础单词关
- Python关键字关
- 符号输入关

每关难度逐渐提升。

例如：

#### 第1关：基础英文词

- print
- input
- list
- string

#### 第2关：Python关键字

- if
- else
- elif
- for
- while

#### 第3关：符号与短代码

- `()`
- `[]`
- `{}`
- `print()`
- `if:`

------

### 5. 结果结算页

每局结束后显示：

- 分数
- 正确数
- 错误数
- 正确率
- 最长连击
- 再来一局按钮

这样用户有成就感。

------

### 6. 本地进度保存

使用 `localStorage` 保存：

- 最高分
- 已解锁关卡
- 最近游玩模式
- 总练习次数
- 每个模式最好成绩

这样用户每次回来都能继续玩。

------

### 7. 手机端适配

虽然手机打字和键盘练习不是完全一致，但手机端也要能玩。

手机端重点：

- 按钮大
- 输入框大
- 布局纵向
- 掉落速度适中
- 字体清晰

------

# 六、第一版不要做的功能

这些先不要碰：

- 登录注册
- 联网排行榜
- 多人对战
- 皮肤商城
- 音效系统
- 背景音乐开关面板
- 成就墙

- 复杂剧情模式

MVP 先做最小闭环。

# 八、玩法设计细化

## 模式1：代码雨

### 规则

- 屏幕上方随机出现单词
- 单词缓慢下落
- 用户在输入框中输入对应内容
- 输入正确后按回车消除
- 单词落到底部则扣血
- 血量归零游戏结束

### 目标

- 打掉尽可能多的目标
- 在有限血量内拿高分

------

## 模式2：快速输入

### 规则

- 一次只显示一个目标
- 用户尽快输入
- 按速度和正确率评分

### 目标

- 提升输入反应速度

------

## 模式3：符号挑战

### 规则

- 连续给出代码符号组合
- 用户快速输入
- 错一个立即提示

### 目标

- 练习括号、引号、冒号、比较符

# 十二、核心交互逻辑

## 代码雨模式

1. 开始游戏
2. 每隔一段时间生成一个目标
3. 目标向下移动
4. 用户输入并回车
5. 匹配成功则消除并加分
6. 落底扣血
7. 血量为 0 结束

## 冲刺模式

1. 显示目标
2. 用户输入
3. 判断对错
4. 记录耗时
5. 显示下一题

## 符号模式

1. 给出符号组合
2. 用户输入
3. 判断
4. 记录正确率

------

# 十三、评分规则建议

第一版评分简单点：

## 代码雨模式

- 正确输入：+10
- 连击额外加分：+2 ~ +10
- 目标漏掉：-1 血

## 冲刺模式

- 正确：+10
- 速度快：额外加分
- 错误：不加分

## 符号模式

- 正确：+15
- 连续正确：加连击

# 十四、UI方向建议

风格建议：

- 学习感 + 游戏感结合
- 清爽，不花哨
- 颜色反馈强
- 动效简单直接

## 色彩建议

- 主色：蓝色
- 正确：绿色
- 错误：红色
- 警告：橙色
- 背景：浅灰 / 深色都可以

## 视觉重点

- 输入框必须明显
- 掉落元素要醒目
- 分数和血量要一眼看到
- 正确/错误反馈要明显

# 十五、MVP验收标准

只要满足下面这些，就算 MVP 成功：

1. 双击 `index.html` 可直接运行
2. 首页可选择模式
3. 至少有一个完整可玩的模式
4. 输入正确能即时加分
5. 输入错误或漏掉会有惩罚
6. 游戏结束有结算页
7. localStorage 能保存最高分和关卡进度
8. 手机端浏览器可正常使用

# 十九、一句话总结

这个项目的 MVP 本质上是：

**一个专门为 Python 初学者设计的打字闯关小游戏，用游戏化方式帮助他们练习键盘输入、熟悉 Python 单词和代码符号。**

它最重要的不是炫，而是：

- 打开就能玩
- 玩的时候确实在练输入
- 玩完以后对写代码更熟一点
